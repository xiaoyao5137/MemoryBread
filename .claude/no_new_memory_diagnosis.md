## 根本原因

### 用户行为分析

**04:36 - 生成工作周报**（knowledge_id=108）
- 活动：生成本周工作周报
- 内容：工作产出和项目进展

**05:17 - 05:27 - 查看工作记录**（被合并到 108）
- 活动：查看 "当前生成的本周工作记录" 文档
- 内容：工作记录、产出、进展

### 为什么被合并？

1. **语义相似**：都是关于"工作记录/周报"
2. **时间接近**：相隔 1 小时（在 24 小时窗口内）
3. **内容重复**：都在处理同一份工作记录文档

**系统判断**：这是同一个工作任务的延续，应该合并

## 为什么多屏采集没有帮助？

### 预期效果

多屏采集应该能解决"标题和内容不匹配"的问题：
- 副屏：VSCode "工作记录"
- 主屏：Gmail / 其他内容
- 拼接后：同时包含两个屏幕

### 实际情况

**问题不在于采集不全，而在于工作内容本身就是重复的**

即使采集了所有屏幕，如果用户一直在做同一件事（查看工作记录），系统仍然会判断为重复。

## 如何生成新的情节记忆？

### 方式 1：做不同的工作（推荐）

进行一些**新的、独特的**工作，例如：

1. **编写新功能代码**
   - 打开不同的文件（不是工作记录）
   - 编写新的函数或模块
   - 提交 Git commit

2. **技术调研**
   - 查看技术文档
   - 搜索解决方案
   - 记录调研笔记

3. **会议讨论**
   - 参加会议
   - 讨论新项目
   - 记录会议纪要

4. **代码审查**
   - 查看 PR
   - 提出修改建议
   - 讨论技术方案

### 方式 2：等待时间窗口过期

等待 24 小时后，即使是相似的工作，也会生成新的记录：

```
04:36 - 生成工作周报（knowledge_id=108）
↓ 等待 24 小时
次日 05:00 - 查看工作记录 → 生成新记录（不会合并）
```

### 方式 3：降低相似度阈值（不推荐）

临时降低阈值，让更多记录被判定为"不相似"：

```python
# extractor_v2.py
threshold = 0.85  # 原值 0.72
```

**缺点**：会产生大量重复记录
