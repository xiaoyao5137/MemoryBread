### 2. RAG召回逻辑修复 (ai-sidecar/creation/service.py)

#### 问题根源
1. **关键词提取过于宽泛**：所有2字以上词都被提取
2. **召回策略过松**：多个关键词用OR连接，匹配任意一个即可
3. **相关性评分过低**：基线0.45，容易误召回不相关文档

#### 修复方案

**A. 扩展停用词表（第599-612行）**
```python
stop = {
    "帮我", "生成", "一份", "关于", "根据", "参考", "文档", "内容", "格式", "需要", "本次",
    "技术", "方案", "建设", "实施", "系统", "平台", "设计", "开发", "项目"  # 新增高频泛化词
}
```

**B. 改进召回逻辑（第311-321行）**
- **旧逻辑**：OR连接，匹配任意关键词
- **新逻辑**：计数模式，至少匹配一半关键词
```python
# 旧：(kw1 OR kw2 OR kw3 OR kw4)
# 新：(match_count >= 2) 当有4个关键词时
```
- **效果**："分销团长技术方案"提取为["分销","团长"]，要求至少匹配1个

**C. 提高相关性阈值（第611-625行）**
- 无关键词时从0.45降到0.35
- 计算公式从 `hits/total * 0.75` 改为 `hits/total`（满分1.0）
- **新增硬阈值**：score < 0.4 直接返回0.0
- 标题命中权重从0.08提升到0.12

#### 修复效果预期
- ✅ 减少无关文档召回（如LangBridge、灵机agent等）
- ✅ 提高高相关度文档排序
- ✅ "分销团长"等领域特定词不会被误匹配到"技术方案"泛类

## 📋 使用说明

### 左侧配置面板
1. 默认折叠状态，点击 `→` 按钮展开
2. 展开后可配置文档类型、权重等参数
3. 点击 `←` 按钮收起

### 参考资料
1. 默认折叠，点击标题区域展开
2. 展开后显示召回的参考文档
3. 每个文档显示权重、相关性分数、推荐原因
4. 再次点击标题区域收起

### RAG召回优化
- 系统会自动过滤高频泛化词
- 要求至少匹配一半关键词才会召回
- 相关度过低的文档会被直接排除
