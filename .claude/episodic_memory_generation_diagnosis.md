## 补充：合并逻辑说明

当前系统的合并逻辑：
1. 新提炼的情节记忆会与已有记录计算相似度
2. 如果相似度 >= 0.80（阈值），判定为重复
3. 重复记录会被合并到已有记录，只更新 `occurrence_count`
4. **不会更新其他字段**（包括 work_item）

这个设计是合理的，避免重复记录。但对于测试新功能，需要生成**不相似**的新内容。
