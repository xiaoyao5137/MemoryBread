# 云端资产快照链路

本文记录 MemoryBread 客户端资产数据的云端备份/恢复链路。该链路用于配置项、知识、操作、文档、时间线等本地资产，不上传原始采集正文、截图路径、输入文本或音频文本。

## 数据范围

快照包含：

- 配置与隐私规则：`user_preferences`、`app_filters`、`app_blacklist`、`privacy_filters`
- 操作与使用资产：`action_logs`、`rag_sessions`、`scheduled_tasks`、`task_executions`
- 记忆资产：`timelines`、`bake_knowledge`、`bake_sops`
- 文档资产：`bake_documents`、`bake_document_sections`、兼容期空表/旧表
- 创作历史：`creation_history`

快照不包含：

- `captures` 原始记录正文
- `ax_text`、`ocr_text`、`input_text`、`audio_text`
- `screenshot_path`、`url`、`webpage_title`、`screenshot_source`
- `vector_index`、FTS shadow tables、系统监控与清理日志

`timelines.capture_id` 仍需要外键，因此快照会导出 redacted capture refs。恢复时这些 refs 会写成 `event_type='snapshot_ref'` 且 `is_sensitive=1` 的占位行，只保留 id/时间/应用窗口元数据，不恢复采集正文或截图。

## 客户端本地 API

- `POST /api/snapshots/assets/export`
  - 导出明文资产快照 JSON 到本机备份目录。
  - 返回 `file_sha256`、文件大小与 manifest。
- `POST /api/snapshots/assets/import`
  - 默认 `dry_run=true`，不会写库。
  - 显式传 `dry_run:false` 时追加/合并导入，不清表。
- `POST /api/snapshots/cloud/backup`
  - 在客户端用 AES-256-GCM 加密资产快照。
  - 调用 admin 上传 URL，PUT 到 OSS，再提交快照元数据。
  - 未传 `recovery_key_base64` 时会生成一次性恢复密钥并返回，调用方必须保存。
- `POST /api/snapshots/cloud/restore`
  - 下载 OSS 对象，核对数据库登记的 size/checksum。
  - 用恢复密钥解密，写出本地明文快照文件。
  - 默认不导入；传 `import_to_local:true` 后仍默认 dry-run。

## 服务端 API

- `POST /v1/snapshots/upload-url`
  - 校验登录用户与设备归属。
  - 生成 scoped OSS object key 和 15 分钟 PUT 签名 URL。
- `POST /v1/snapshots`
  - 上传完成后登记快照元数据。
  - 同一用户同一 `oss_object_key` 重复提交时，如果 size/version/checksum 完全一致，直接返回已有快照。
  - 若元数据不一致，返回 `SNAPSHOT_OBJECT_CONFLICT`。
- `GET /v1/snapshots`
  - 返回快照版本、对象 key、checksum 与提交状态，用于核对 RDS 元数据。
- `GET /v1/snapshots/:snapshot_id/download-url`
  - 返回 10 分钟 GET 签名 URL，并附带登记的 size/checksum。

## 重复判断

本地导入：

- 普通资产表按主键 `INSERT OR IGNORE`，同一快照重复导入不会重复插入。
- 配置类表按 `key` / `app_name` / `bundle_id` / `filter_type` upsert。
- `capture_refs` 使用 capture id 去重，不覆盖真实 capture。

云端登记：

- OSS object key 由服务端按用户、设备和 UUIDv7 生成。
- `snapshots(user_id, oss_object_key)` 有唯一索引。
- 完成登记先查已有对象；一致则幂等返回，不再重复写 outbox。
