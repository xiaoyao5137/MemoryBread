# AX 缓存优化：避免重复检测

## ✅ 已实现

### 1. 应用级 AX 支持缓存

**问题**：VSCode 每 60 秒都要执行快速检测（50ms），即使已知不支持 AX

**解决**：添加应用级缓存，记录哪些应用不支持 AX

```rust
// 缓存结构
static AX_SUPPORT_CACHE: OnceLock<Mutex<HashMap<String, (bool, Instant)>>> = OnceLock::new();
const AX_CACHE_TTL_SECS: u64 = 3600; // 缓存 1 小时

// 检测流程
fn check_ax_support(app_name: Option<&str>) -> bool {
    // 1. 检查缓存
    if let Some((supported, timestamp)) = cache.get(app_name) {
        if timestamp.elapsed() < 1小时 {
            return *supported; // 直接返回缓存结果
        }
    }
    
    // 2. 执行检测（50ms）
    let supported = osascript_check();
    
    // 3. 更新缓存
    cache.insert(app_name, (supported, now));
    
    supported
}
```

| **VSCode 后续检测** | 50ms | <1ms | **50 倍** |

### 3. 完整流程

```
VSCode 采集周期 1:
├─ AX 快速检测: 50ms (未命中缓存)
├─ 检测结果: 不支持
├─ 缓存写入: ("Code", false, now)
└─ 降级 OCR: 1.8秒

VSCode 采集周期 2-60:
├─ AX 快速检测: <1ms (命中缓存)
├─ 检测结果: 不支持 (来自缓存)
└─ 降级 OCR: 1.8秒

总耗时:
- 优化前: 60 × 50ms + 60 × 1.8s = 3s + 108s = 111秒
- 优化后: 50ms + 59 × <1ms + 60 × 1.8s ≈ 50ms + 108s = 108.05秒
- 节省: 2.95秒/小时
```

### 关键优势

1. **避免重复检测** - VSCode 只检测一次，后续直接用缓存
2. **无需黑名单** - 自动学习哪些应用不支持 AX
3. **缓存自动过期** - 1 小时后重新检测（应对应用更新）
4. **线程安全** - 使用 Mutex 保护缓存
